News Search Engine



Category: Domain Specific Information Retrieval System

Document Source: The Hindu Times archives

Overview about the project

A News Search Engine was implemented using the Vector Space Model.

Vector Space model is a category of Ranked Retrieval Model which uses cosine similarity between the query and the document to retrieve results.

lnc.ltc weighting scheme was used.

> Query weighing: Logarithmic tf (term frequency), idf (Document frequency), no cosine normalisation.

> Document weighing: Logarithmic tf (term frequency), no idf (Document frequency), cosine normalisation.



Novelty Feature

  1. The News Search comes in an attractive, simple to use GUI that provides the user with the feature to find the required data in microseconds. The user if wished would also be redirected to the original source of the data at www.thehindu.com

The GUI was made in Python from the PyQt5 Library.

  1. The News search also gives relevant to the query, few more suggestions tagged as ‘You may also like’ has been also implemented.

  1. The time taken to retrieve the search result is also printed.



Task 1: Data Scraping

To retrieve a document corpus using web scraping tools from any relevant website

Task 2: Document Processing

To process the document corpus using tokenisation, stemming and a document weighing scheme.

Task 3: Query Processing

To process a query using a weighing scheme and getting the cosine similarity scores with all documents to get the best results.

    Step 2- Data Processing

    Packages Used-

  • os
  • nltk- Natural Language Tokenizer
  • re- Regliar Expression
  • pickle

    • (data_proc.py)

    Data Processing starts with converting the whole text body of individual articles into Lower Case. And tokenization is achived using nltk package. Having removed all the special characters and numbers; the final text contains no stop words and Stemming is done. For the purpose of stemming porter_stemmer is used. For each individual document in the corpus a dtf Dictionary- python data_structure is generated which holds the frequnecy of individual terms in the document. The inverse frequency of each term in the documentm is generated in idf_desc.pickle file by iterating over whole corpus. Final Output are two pickle files:

  • doc_desc.pickle- contains the file decriptor (filename, link, document- term frequency
  • idf_desc.pickle- contains the idf of each term in the vocabulary as list of dictionary
